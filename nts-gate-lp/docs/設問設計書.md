# 無料診断（設問）設計書

**参照:** `NTS_ゲートLP_設計書.md` §1 トンマナ・§5 診断ツール遷移仕様  
**ルート:** `/diagnosis`（LP の「1分で無料診断する」等から遷移）  
**方針:** 個人情報は取得しない（メール等は v2 以降で任意検討）

---

## 1. 目的・ゴール

| 項目 | 内容 |
|------|------|
| 目的 | 補助金の対象になりやすいかの**粗い整理**、および**エンドユーザー（自社利用）**と**パートナー候補（顧客への提案・提携意欲）**の切り分け |
| ゴール | 回答に応じた結果画面へのルーティングと、GA4 `diagnosis_step` / `diagnosis_result` 計測 |

---

## 2. 設問数（4問）

- **4問（各4択）**で、利用者タイプ・規模・課題・時期/準備度をカバー。  
- LP の「3つの質問」表記とは異なるため、LP 側のマイクロコピーは「**約1分・4つの質問**」等へ揃えることを推奨。

---

## 3. ユーザーフロー

```
CTA「1分で無料診断する」クリック
  → /diagnosis
  → ウィザード（1/4 … 4/4）
  → 結果ページ（/result/end-user または /result/partner）
```

回答内容は **sessionStorage**（キー `nts_diagnosis_v1`）に保存し、結果ページで表示に利用する。

---

## 4. UI 仕様

- 上部: プログレスバー（現在 / 全4）  
- 中央: 1問1画面、見出し＋補足  
- 選択肢: カード型4択（単一選択）  
- 下部: 「戻る」「次へ」（最終は「結果を見る」）  
- a11y: ラジオグループ相当のキーボード操作、`fieldset` / `legend` を使用

---

## 5. 設問定義

### Q1【分岐の核】あなたの会社にとって、いま一番近いのはどれですか？

| 選択肢 | 内部値 | 結果ルーティングの目安 |
|--------|--------|-------------------------|
| 自社の投資・採用・設備などに、補助金を使いたい | `end_user` | `/result/end-user` |
| 取引先や顧客の提案に、補助金情報を組み込みたい | `partner_sales` | `/result/partner` |
| 自社利用もありつつ、顧客向けにも情報提供したい | `hybrid` | `/result/end-user` |
| まだ方針が決まっていない・情報収集のみ | `exploring` | `/result/end-user` |

### Q2【規模】従業員規模に近いのはどれですか？

| 選択肢 | 内部値 |
|--------|--------|
| 5名以下 | `emp_1_5` |
| 6〜20名 | `emp_6_20` |
| 21〜100名 | `emp_21_100` |
| 101名以上・または該当しない | `emp_101_plus` |

### Q3【課題】いま最も関心が高いテーマはどれですか？

| 選択肢 | 内部値 |
|--------|--------|
| 人手不足・採用・省力化 | `theme_workforce` |
| 設備投資・工場・店舗の更新 | `theme_capex` |
| IT・DX・業務効率化 | `theme_dx` |
| 事業承継・組織再編・その他 | `theme_succession_other` |

### Q4【行動時期】※ Q1 で文言が切り替わる

**Q1 が `partner_sales` 以外のとき（エンドユーザー・ハイブリッド・探索）**

| 選択肢 | 内部値 |
|--------|--------|
| 3か月以内に動きたい | `timing_3m` |
| 半年以内 | `timing_6m` |
| 1年以内・長期で検討 | `timing_1y` |
| まだ時期は決めていない | `timing_tbd` |

**Q1 が `partner_sales` のとき**

| 選択肢 | 内部値 |
|--------|--------|
| すでに顧客へ提案している | `partner_active` |
| 半年以内に提案体制を整えたい | `partner_6m` |
| 情報提供・勉強会レベルから始めたい | `partner_light` |
| まず仕組み・提携内容を知りたい | `partner_learn` |

---

## 6. ロジック・結果

- **遷移先:** `q1 === partner_sales` → `/result/partner`、それ以外 → `/result/end-user`  
- **免責:** 結果画面に「本診断は目安であり、申請可否は各公募要領での判断が必要です」を表示する。

---

## 7. 計測

- 各ステップ完了時: `diagnosis_step`（`step`: 1〜4, `action`: `complete`）  
- 結果遷移直前: `diagnosis_result`（`segment`: `q1` の内部値）
